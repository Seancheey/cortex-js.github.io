    function mathlive.getOriginalContent(element: string | HTMLElement | MathField, options?): string

    After calling renderMathInElement or makeMathField the original content can be retrived by calling this function.

    Given the following markup:

    <span id='equation'>$$f(x)=sin(x)$$</span>

    The following code:

    MathLive.renderMathInElement('equation');
console.log(MathLive.getOriginalContent('equation'));

    will output:

    $$f(x)=sin(x)$$
    element: string | HTMLElement | MathField
    A DOM element ID, a DOM element or a MathField.
    options?
    options.namespace?: string
    →: string
    the original content of the element.

    function mathlive.latexToMarkup(text: string, mathstyle: string, format?: string): string

    Convert a ‘‘LaTeX’’ string to a string of HTML markup. It’s easy.

    text: string
    A string of valid LaTeX. It does not have to start with a mode token such as $$ or \(.
    mathstyle: string
    If 'displaystyle' the “display” mode of TeX is used to typeset the formula, which is most appropriate for formulas that are displayed in a standalone block. If 'textstyle' is used, the “text” mode of TeX is used, which is most appropriate when displaying math “inline” with other text (on the same line).
    format?: string
    →: string

    function mathlive.makeMathField(element: HTMLElement | string, config?: MathFieldConfig): MathField

    Convert a DOM element into an editable math field.

    After the DOM element has been created, the value element.mathfield will return a reference to the mathfield object. This value is also returned by makeMathField

    element: HTMLElement | string
    A DOM element, for example as obtained by document.getElementById(), or the ID of a DOM element as a string.
    config?: MathFieldConfig
    →: MathField
    Given the HTML markup: 
    <span id='equation'>$f(x)=sin(x)$</span>

    The following code will turn the span into an editable mathfield.

    import MathLive from 'dist/mathlive.mjs';
MathLive.makeMathField('equation');

    function mathlive.renderMathInDocument(options?): void

    Transform all the elements in the document body that contain LaTeX code into typeset math.

    Note: This is a very expensive call, as it needs to parse the entire DOM tree to determine which elements need to be processed. In most cases this should only be called once per document, once the DOM has been loaded. To render a specific element, use renderMathInElement()

    See: USAGE_GUIDE

    import MathLive from 'dist/mathlive.mjs';
document.addEventListener("load", () => {
    MathLive.renderMathInDocument();
});
    options?: [key: string]: any
    options..[key: string]: any
    →: void

    function MathField.$applyStyle(style): void

    Apply a style (color, bold, italic, etc…).

    If there is a selection, the style is applied to the selection

    If the selection already has this style, remove it. If the selection has the style partially applied (i.e. only some sections), remove it from those sections, and apply it to the entire selection.

    If there is no selection, the style will apply to the next character typed.

    style
    an object with the following properties. All the properties are optional, but they can be combined.
    style.backgroundColor?: string
    style.color?: string
    style.fontFamily?: string
    style.mode?: string
    Either 'math', 'text' or ‘command’
    style.series?: string
    style.shape?: string
    style.size?: string
    The font size: ‘size1’…‘size10’ ‘size5’ is the default size
    →: void

    function MathField.$insert(s: string, options?): void

    This method can be invoked as a selector with $perform("insert") or called explicitly.

    It will insert the specified block of text at the current insertion point, according to the insertion mode specified.

    After the insertion, the selection will be set according to the selectionMode.

    s: string
    The text to be inserted
    options?
    options.feedback: boolean
    If true, provide audio and haptic feedback
    options.focus: boolean
    If true, the mathfield will be focused after the insertion
    options.format: “auto” | “latex”
    The format of the string s:
    • 'auto': the string is interpreted as a latex fragment or command) (default)
    • 'latex': the string is interpreted strictly as a latex fragment
    options.mode: “text” | “math” | “”
    ‘text’ or ‘math’. If empty, the current mode is used (default)
    options.resetStyle: boolean
    If true, the style after the insertion is the same as the style before (if false, the style after the insertion is the style of the last inserted atom).
    options.selectionMode: “placeholder” | “after” | “before” | “item”
    Describes where the selection will be after the insertion:
    'placeholder' the selection will be the first available placeholder in the item that has been inserted (default)
    'after' the selection will be an insertion point after the item that has been inserted
    'before' the selection will be an insertion point before the item that has been inserted
    'item' the item that was inserted will be selected
    →: void
